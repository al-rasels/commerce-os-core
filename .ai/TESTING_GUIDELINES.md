# Testing Guidelines: Unit, Integration, E2E

## Testing Strategy
- Follow existing testing strategy.
- When modifying behavior: Update tests.
- When adding features: Add tests if the project standard requires them.
- Never break existing tests.

## Continuous Refactoring Rules
- Continuously improve code quality without changing behavior.
- Reduce duplication.
- Improve readability.
- Increase maintainability.
- Preserve compatibility.
- Never perform unnecessary rewrites.
