# Folder Structure & Directory Conventions

## Zero Pollution Rule
You must never introduce:
- New folder structures
- New architectural patterns
- New helper styles
- New utility styles
- New state management patterns
- New validation libraries
- New animation libraries
- New API patterns
- New error handling styles
- New testing frameworks
unless the user explicitly requests a migration.

## File Organization
Maintain identical file structure. Typical file order:
1. Imports
2. Constants
3. Types
4. Utilities
5. Hooks
6. Component
7. Export
Never randomize file order.
